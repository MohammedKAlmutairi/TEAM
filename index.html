Team Formation Problem

Facilitating the formation of Diverse Teams based on Usersâ€™ Feedback

Abstract

Team recommender algorithms assist users in finding teams by leveraging users' characteristics, skills, and relationships. Despite recent advancements, most existing team formation algorithms do not consider user preferences when searching for optimal team combinations. Moreover, most of them do not consider when multiple users search for and choose others for their teams, which changes the possible number of team combinations. In this work, we formulate the team formation problem as a multi-armed bandit problem. Our approach sequentially selects teams from multiple pre-formed team combinations to find which ones are the most liked by users. The selection is based on contextual information about both the users and teams, while adapting its team selection strategy based on user feedback to maximize overall user satisfaction. The contributions of this work are twofold. First, we propose a novel application of a contextual bandit algorithm to the domain of team formation. Second, through extensive experiments on simulated data from different real-world datasets, we demonstrate how our proposed algorithm learns from user feedback significantly enhances the relevance of team recommendations compared to benchmark algorithms.

This repository includes the following resources:

  • The pre-processed dataset utilized in this study.
  • The Python script for generating the non-dominant team based on the Pareto principle, using the Non-dominated Sorting Genetic Algorithm II (NSGA-II), including a non-dominated team chart for each dataset case examined in the study.
  • The Python code for executing the proposed algorithm alongside the baseline algorithms, which include:
    • Proposed Algroithm B-UCB
    • UCB
    • Thompson Sampling
    • ε-greedy algorithm
    • Greedy algorithm
    • Random selection
  • The Python code for the simulation evaluation discussed in Section <ref>evaluation</ref>.
  • The R code for implementing the Gittins Index (GI) as explained in the study <cite>edwards2019practical</cite>, including the output files.
